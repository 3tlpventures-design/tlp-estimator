# Reference data — Wayne's TLP Master Pricing Table

This directory holds **reference artifacts** — files maintained outside this
repo that we keep a snapshot of for context. The estimator does NOT read
them at runtime. Only `data/Allowance_Pricing_Levels.xlsx` and
`data/BLANK_Allowances_3_5_2024.xlsm` feed `pricing.json`.

---

## `TLP MASTER PRICING TABLE.xlsx`

Wayne's (Dad's) pricing-intelligence workbook. Maintained programmatically
by another Claude Code instance ("Cody") on Wayne's machine via Python +
gspread. Snapshot date: April 2026.

### Why it's here

1. **Coordination.** Two AIs are working on TLP's pricing system. This repo
   is the customer-facing estimator. Wayne's workbook is the underlying
   cost-intelligence system. They're different systems that need to meet
   in Phase 2+ — keeping a snapshot here means this repo can reason about
   his schema.
2. **Reference for future phases.** When we build the bridge between his
   actual-paid cost data and our tier-based customer allowances (see
   `future-plans.md`), we'll need his column definitions, Flex Rules, and
   Cost Type taxonomy in front of us.
3. **Archival.** His workbook is active and changes often. This is a
   point-in-time copy of what his system looked like when we first saw it.

### Relationship to our pricing

| | Wayne's workbook (this file) | Our `pricing.json` |
|---|---|---|
| Shape | Invoice rows × cost columns | 79 categories × 5 tiers |
| Answers | "What did a 2×12 SYP cost on Kupin, Aug 2025?" | "What's a Tier 3 vanity light worth?" |
| Source | Real paid invoices | TLP historical allowance rates |
| Keys | Trade + Scope + Vendor + Date | `category_id` (e.g. `light-chandelier-low`) |
| Dimensions | Division, Phase, Trade, Cost Type, Confidence, Flex % | Tier (t1–t5), `tlpFixed` |
| Scale | Invoice-level (thousands of rows over time) | Category-level (79 rows) |

**These are not the same table.** They serve different purposes. His is
the backend cost-basis database. Ours is the front-end tier allowance
table. Future work (see `future-plans.md` Phase 3+) connects them.

### Rules for this directory

- **Do not import this file into the build pipeline.** `build_pricing.py`
  only reads `data/Allowance_Pricing_Levels.xlsx` and `BLANK_Allowances_*`.
- **Do not edit this file here.** It is a snapshot. Wayne's working copy
  lives in his Dropbox + on his machine where Cody maintains it.
- **Refresh the snapshot when the schema changes.** If Wayne adds tabs,
  renames columns, or substantially changes Flex Rules, replace this file
  and note the change in `CHANGELOG.md`.
